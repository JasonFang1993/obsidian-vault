收到了！这篇是 **sessions_yield** 的官方文档 🦞

**核心就一句话**：主代理派完子代理后，主动 `sessions_yield()` 结束轮次，等子代理结果自动推送过来，不用再傻傻轮询了。

**解决的问题**：
- Token 浪费（轮询太烧钱）
- 模型不听话（某些模型会忽略"别轮询"的指令）
- 超时丢结果（ANNOUNCE_EXPIRY_MS 太短）

**使用简单**：
```javascript
sessions_spawn({ agentId: "xxx", task: "..." })
sessions_yield({ message: "等一会儿..." })
// 完事儿，结果自动送达
```

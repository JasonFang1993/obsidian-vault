### 问题：AI Agent 为什么总是"失忆"

大部分 AI Agent 有个致命问题：每次对话都是全新开始。你今天教它一个规则，明天它就忘了。你纠正过的错误，下次还会犯。

这不是模型的问题，是架构的问题。传统 Agent 把所有上下文都塞进 prompt，session 一结束，记忆就清空了。

Shubham 的方案不一样：把记忆写进文件系统。Agent 每次启动都读这些文件，相当于有了"长期记忆"。40 天后，同样的模型，输出质量天差地别。

### 核心架构：三层文件系统

Shubham 的 Agent 系统只有三层，全是 Markdown 文件：

第一层：身份层（Identity）

SOUL.md：Agent 是谁，性格是什么IDENTITY.md：快速参考卡（名字、角色、emoji）USER.md：用户是谁，有什么偏好

第二层：操作层（Operations）

AGENTS.md：行为规则，每次启动读什么文件HEARTBEAT.md：自检规则，定时检查系统健康专业指南：针对具体任务的详细规则

第三层：知识层（Knowledge）

MEMORY.md：长期记忆，重要经验和教训memory/YYYY-MM-DD.md：每日日志，原始记录shared-context/：跨 Agent 共享的知识

没有数据库，没有消息队列，没有复杂框架。文件系统就是集成层。

### 第一层：给 Agent 一个灵魂

### SOUL.md：定义性格

Shubham 给每个 Agent 起了电视剧角色的名字。比如他的研究 Agent 叫 Dwight（《办公室》里那个较真的销售）：

Dwight — 研究大脑。以 Dwight Schrute 命名，因为你们有同样的特质：极度认真，对领域内的一切了如指掌，把工作当命。不废话，不猜测，只要事实和来源。

1绝不编造：每个结论都要有来源链接2信号大于噪音：不是所有热点都值得关注3不确定就标记 [UNVERIFIED]

为什么用电视角色？因为 Claude 的训练数据里有这些角色的完整人设。你说"Dwight Schrute 的能量"，模型立刻就懂那是什么风格。相当于免费加载了 30 季的性格设定。

关键原则：SOUL.md 要短。控制在 60 行以内，因为它每次 session 都要加载。太长会挤占实际工作的上下文空间。

SOUL.md 模板：

[Agent 名字] — [一句话描述]。[可选：参考的电视角色]

[这个 Agent 具体做什么。要具体，不要泛泛而谈]

1. [最重要的规则]

2. [第二重要的规则]

3. [第三重要的规则]

[这个 Agent 和谁协作？谁会用它的输出？]

### IDENTITY.md：快速参考卡

SOUL.md 是完整人格，IDENTITY.md 是名片。名字、角色、风格、一句话。

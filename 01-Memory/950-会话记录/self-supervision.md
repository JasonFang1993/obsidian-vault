# 自我监督机制

## 监督触发器

**每次心跳时自动执行**：
1. 读取当前 state 文件
2. 检查 pending 任务
3. 执行任务
4. 写入执行结果
5. 如果有未完成，标记原因

---

## 每日自检清单

| 检查项 | 标准 | 触发动作 |
|--------|------|----------|
| 计划任务 | state 里有 pending？ | 立即执行 |
| 用户反馈 | 有新反馈未记录？ | 写入 feedback.md |
| 版本号 | 超过 7 天没更新？ | 触发反思 |
| 学习目标 | 超过 14 天没学新东西？ | 触发学习 |

---

## 监督日志格式

```json
{
  "last_check": "2026-03-17T18:00:00+08:00",
  "check_result": {
    "pending_tasks": 2,
    "feedback_count": 0,
    "version_age_days": 0,
    "learning_gap_days": 0
  },
  "actions_taken": [],
  "issues": []
}
```

## 检验方法

1. **你问我答**：你可以随时问我"检查一下你的计划"
2. **心跳触发**：每 55 分钟自动检查
3. **文件可查**：所有状态都在 heartbeat-state.json

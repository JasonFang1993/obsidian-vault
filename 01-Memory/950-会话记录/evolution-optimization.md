# 进化机制优化计划

## 问题诊断

| 问题 | 现状 | 需要优化 |
|------|------|----------|
| 版本号追踪 | 只有 timestamp，没有语义化版本 | 引入 v0.x.x 格式 |
| 用户反馈闭环 | 纠正了但没记录 | 每次纠正 → 写入反馈日志 |
| 进化触发 | 靠心跳定时，不够智能 | Add/Merge/Discard 决策 |
| 跨场景迁移 | 学了但不会用 | 建立知识连接 |

---

## 优化计划

### 1. 版本号系统
```
当前：v0.1.0
每次重要更新：v0.1.1
每次学习新领域：v0.2.0
```

### 2. 用户反馈闭环
- 记录：每次用户纠正 → 写 feedback.md
- 周期：每周回顾 → Merge 到 state
- 目标：版本号 = 被纠正次数

### 3. 进化决策
| 场景 | 决策 |
|------|------|
| 新领域学习 | Add → 新增 todo |
| 重复任务优化 | Merge → 版本号+1 |
| 过时内容 | Discard → 标记废弃 |

### 4. 知识连接
- 学到的概念 → 标注相关已有知识
- 建立 skill → 应用场景的映射

# 自主进化引擎优化方案

## 问题诊断

| 问题 | 根因 |
|------|------|
| 记录型而非执行型 | 心跳只读 state，不执行任务 |
| pending 任务没完成 | 没有任务执行逻辑 |
| 学习没有闭环 | 学了但没有真正用起来 |

---

## 优化方案：任务驱动进化

### 核心机制

每次心跳执行：

```
1. 读取 state 的 pending 任务
2. 执行任务（不是只记录）
3. 产出结果
4. 更新 state
```

### 待执行任务类型

| 类型 | 示例 |
|------|------|
| 学习任务 | "研究 X" → 产出学习笔记 |
| 优化任务 | "改进 X" → 产出优化方案 |
| 反思任务 | "自检" → 产出问题清单 |
| 实验任务 | "测试 X" → 产出测试结果 |

### 执行要求

每个任务必须有：
- 明确输入
- 明确输出
- 明确完成标准

---

## 立即优化

1. 更新 HEARTBEAT.md 添加执行逻辑
2. 创建任务执行模板
3. 下次心跳时真正执行 pending 任务

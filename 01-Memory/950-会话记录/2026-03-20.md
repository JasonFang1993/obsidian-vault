### 核心要点

#### AGENTS.md 最佳实践
- 放在 workspace 根目录，每次新 session 自动加载
- 核心结构：Every Session 启动流程 → 记忆分层 → 安全边界
- 启动顺序：SOUL.md → USER.md → memory/YYYY-MM-DD.md → MEMORY.md（仅主 session）
- 记忆分层：MEMORY.md(索引) / memory/projects.md / memory/lessons.md / memory/YYYY-MM-DD.md
- 记结论不记过程，结构化日志检索命中率高

#### memoryFlush 配置
```json
"compaction": {
  "reserveTokensFloor": 20000,
  "memoryFlush": {
    "enabled": true,
    "softThresholdTokens": 4000
  }
}
```

#### SiliconFlow bge-m3 免费 embedding（国内方案）
- model: BAAI/bge-m3
- baseUrl: https://api.siliconflow.cn/v1
- 完全免费，中英文双语支持好

#### 子 Agent 模型分级策略
| 等级 | 模型 | 适用场景 |
|------|------|----------|
| 🔴 高 | opus | 复杂架构、多文件重构 |
| 🟡 中 | sonnet | 写代码、信息整理（默认）|
| 🟢 低 | haiku | 搜索汇总、简单操作 |

#### Cron 关键踩坑
- 必须设 tz 字段！不设默认 UTC，"早上9点"变北京时间下午5点
- sessionTarget: main → systemEvent
- sessionTarget: isolated → agentTurn

#### Skill 触发核心
- description 字段决定触发率，越具体越好
- 中英文触发词都要写

#### 多渠道格式差异
- Discord: 支持 Markdown/表格/代码块
- Telegram: 支持 Markdown，不支持表格
- WhatsApp: 不支持表格，用 bullet list

## 🔥 学到的关键认知

### OpenClaw 9+10篇系列总结

#### 核心架构
- Gateway = 消息网关 + Agent运行时 + 调度器 + 设备管理
- 单进程设计，18789端口
- 消息链路：Channel接收 → 路由匹配 → 队列 → Session → Agent → 回复

#### 记忆系统（四层）
1. 文件记忆：MEMORY.md + 每日日记
2. 自动刷盘+压缩：memoryFlush先刷盘再压缩
3. 向量搜索：Hybrid Search（BM25 + 向量）
4. 高阶方案：LanceDB/Supermemory

#### 多Agent协作
- Spawn：异步后台任务，不阻塞主Session
- 独立Agent：有独立workspace/记忆/Session
- 我们模式：main + OpenCode子Agent

#### 安全分层
- Gateway认证 → Agent工具权限 → Workspace隔离 → 沙箱

#### 三种调度器
- Heartbeat：周期性巡检（30分钟）
- Cron：精确时间任务（分钟级）
- Webhook：外部触发

#### 真实工作流案例
1. AI新闻日报：Cron+搜索+视频生成
2. PPT视频：React+Remotion代码生成
3. 公众号写作：main调研+writer写+配图

### 可以借鉴的实践

- 多Agent分工（main+writer）
- 视频生成自动化
